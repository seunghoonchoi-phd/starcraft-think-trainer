# 이중과제 입력·판단 훈련

사용자가 손을 움직이는 동안 새 시각 정보를 보고 다음 행동을 고르면, 두 과제는 행동 선택 단계에서 서로 영향을 줍니다. 이 앱은 그때 입력 속도와 판단 정확도가 얼마나 달라지는지 측정합니다.

이 앱은 전체 입력 수(APM)를 점수로 계산하지 않습니다. 사용자는 숫자 키를 누른 뒤, 이동한 표적을 클릭해야 완료 동작 한 개를 기록합니다. 사용자가 순서를 바꾸거나 잘못된 키를 누르면 앱은 해당 입력을 오류로 기록합니다.

## 10분 측정·훈련 순서

사용자는 훈련을 바로 시작하거나 왼쪽 단계 목록에서 원하는 단계만 연습할 수 있습니다. 사용자가 훈련을 멈추면 앱은 홈으로 돌아가며, 사용자는 결과 화면에서 기록 파일을 저장할 수 있습니다.

1. 입력 기준선: 사용자는 판단 문제 없이 숫자 키와 표적 클릭을 순서대로 수행합니다.
2. 판단 기준선: 사용자는 입력하지 않고 상황 그림을 보고 먼저 처리할 행동을 고릅니다.
3. 동시 수행: 사용자는 세 단계 입력을 계속 수행하면서 그림 판단도 함께 풉니다.
4. 우선 과제 변경: 앱은 입력 우선, 판단 우선, 동일 비중 조건을 번갈아 표시합니다.
5. 판단 규칙 변경: 앱은 시각 상황의 우선순위 규칙을 바꿉니다.
6. 입력 억제: 앱이 빨간 STOP 표적을 표시하면, 사용자는 숫자 키와 클릭을 모두 멈춥니다.
7. 변경 조건 검사: 앱은 숫자 키 순서를 바꿉니다. 앱은 이 구간에서 정답을 바로 알려 주지 않습니다.

앱은 입력 속도 유지율, 판단 성적 유지율, 새 상황 판단 정확도를 따로 표시합니다. 앱은 각 구간의 판단 정확도를 중앙 반응 시간(초)으로 나눠 판단 효율을 계산합니다. 앱은 입력과 판단을 함께 수행한 구간의 판단 효율을 판단 기준선의 판단 효율로 나눕니다. 앱은 이 비율과 판단 정확도 유지율 중 낮은 값을 판단 성적 유지율로 표시합니다. 따라서 사용자가 오답을 빠르게 골라도 앱은 그 결과를 높은 판단 성적으로 표시하지 않습니다. 사용자는 각 측정값을 보고 입력 과제와 판단 과제 중 어느 과제의 성적이 낮아졌는지 확인할 수 있습니다.

## 연구 결과를 훈련 순서에 적용한 방법

인지심리학 연구자들은 사람이 두 과제에 빠르게 반응할 때 두 번째 행동의 선택 시간이 길어질 수 있다고 보고했습니다. 다른 연구자들은 반복 연습이 두 과제를 함께 수행할 때 생기는 성능 저하를 줄일 수 있다고 보고했습니다. 일부 연구자들은 참가자가 과제 순서를 섞어 연습한 뒤 연습 문제와 다른 문제에서 더 높은 성적을 냈다고 보고했습니다. 그러나 실제 현장 과제를 다룬 연구자들은 효과가 작거나 통계적으로 분명하지 않다고 보고했습니다.

이 앱은 사용자가 세 단계 입력을 먼저 연습하게 합니다. 이후 앱은 입력 과제와 그림 판단을 함께 제시하고, 마지막에는 숫자 키 순서를 바꿉니다. 앱은 입력 정확도와 판단 정확도가 기준을 넘을 때만 표적 표시 간격을 줄입니다.

- [Guadagnoli & Lee (2004), Challenge point framework](https://pubmed.ncbi.nlm.nih.gov/15130871/)
- [Czyż et al. (2024), Contextual interference and transfer meta-analysis](https://pmc.ncbi.nlm.nih.gov/articles/PMC11349744/)
- [Glass et al. (2013), Real-time strategy game training](https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0070350)
- [Smith & Basak (2023), Video-game training meta-analysis](https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0285925)

## 이 앱이 확인하지 않은 효과

연구자들은 이 앱이 특정 게임이나 업무의 성과를 올리는지 직접 시험하지 않았습니다. 앱에서 사용자의 입력 속도나 판단 정확도가 높아져도 사용자는 실제 작업에서 행동 누락과 지연이 줄었는지 따로 확인해야 합니다.

이 앱은 사용자의 건강 상태, 지능, ADHD 증상을 검사하거나 치료하지 않습니다.

## 실행과 검사

이 앱은 정적 파일만 사용하므로 별도 빌드가 필요하지 않습니다. 개발자는 로컬 HTTP 서버에서 `index.html`을 열어 앱을 실행할 수 있습니다.

```powershell
npm.cmd test
npm.cmd run check
```

개발자가 `main` 브랜치에 변경사항을 push하면 GitHub Actions가 테스트를 실행한 뒤 GitHub Pages에 앱을 배포합니다. 앱은 사용자의 훈련 기록을 서버로 보내지 않습니다. 브라우저는 훈련 기록을 브라우저 저장 공간(`localStorage`)에만 저장합니다.
